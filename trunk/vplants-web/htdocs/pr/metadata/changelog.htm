vPlants Metadata Change Log

v1.2 - April, 2002

The following changes were prompted by our meeting at UIC on March 7. The general theme in this release was to introduce consistancy in terminology.

For starters, I've renamed the layers in our metadata to better reflect their purpose:

Previous Layer Name New Layer Name
vpindex index
vphdr taxon
vpdtl spm

Layer Change Reason
index Changed <vpindex> to <index> "vp" is unnecessary.
index Broke up scinmlink into 2 elements, <taxon> and <link>. <taxon> is the parent, <link> is the child. <taxon> can have 1 to many <link> elements. See example.

Better terminology and a more flexible element layout.

taxon Changed <vphdr> to <taxon> Better terminology.
taxon Changed <spmlink> to <link>. See example. Consistancy.
spm Changed <vpdtl> to <spm> Better terminology.
spm Changed <typespec> to <typespm> Consistancy.
spm Changed <spmpg> to <homepg>. See example.

Better terminology.

v1.1 - March, 2002

The following changes have been applied to the last version of the vPlants Metadata Standard:

Layer Change Reason
vphdr Changed the following tag names:
SPECIMEN to SPM
SPID to SPMID
SPLINK to SPMLINK
 We felt the shorthand SP for specimen was to brief, especially considering the potential development of species pages. SPM is slightly more specific.
vphdr Introduced minutes (MIN) into the UPDTTM The bot keeps track of when it visits a set of metadata. The UPDTTM field allows the bot to decide if a particular set of specimen data has changed since it's last visit. Introducing minutes provides the potential for the bot running multiple times in the same hour. Perhaps this is overkill, but we felt it would be easier to add it now instead of later.
vpdtl Changed TSHIP to TOWNSHIP Clarity.
vpdtl SITESURVEY - broke out the Township/Range number and bearing into separate fields. This decision keeps with the theme of breaking individual pieces of data into the finest granularity possible.
vpdtl Changed SPECPG to SPMPG As mentioned above, this eliminates any confusion between specimen and species.
vpdtl SPMPG/IMAGE tags include a URI reference.

In the previous version, the IMAGE tag simply indicated the presence of an image and it's type. Now we are also providing the web address for the image via it's URI (i.e., http://www.mortonarb.org/acc123.jpg).

An awareness of "where" the images reside allows us to:
(1). Embed the image within results pages at the portal.
(2). Provide users with "direct links" to the images.
